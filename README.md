# MetalDrive
KI-gestützte Automatisierung des IT-Support-Prozesses

IT-Support heute
Der IT-Support wird von 2 Mitarbeitern nebenbei betreut, ihre Hauptaufgabe ist der Betrieb
der IT-Infrastruktur. Es existiert kein zentrales Ticketsystem. Anfragen kommen über drei
unkontrollierte Kanäle:
• Direkte E-Mail an einen der IT-Mitarbeiter
• Mündlich oder per Microsoft Teams-Nachricht
• Post-it am IT-Schreibtisch

Der Prozess läuft wie folgt ab:
• Anfrage eingeht (E-Mail, Teams, mündlich)
• IT-Mitarbeiter liest oder hört die Anfrage
• Bewertet und priorisiert nach Bauchgefühl
• Bearbeitet das Problem parallel zu laufenden Aufgaben
• Löst das Problem (oder vertagt es)
• Keine systematische Dokumentation des Vorgangs

Reibungsverluste und Schwachstellen
• Doppelarbeit: Beide IT-Mitarbeiter erhalten häufig dieselbe Anfrage parallel
• Keine Priorisierung: Wer zuerst fragt, wird zuerst bedient, unabhängig von
Dringlichkeit
• Kein Wissensaufbau: Gelöste Probleme werden nicht festgehalten, dieselben Fragen
kommen immer wieder
• Unterbrechungskultur: IT-Mitarbeiter werden fortlaufend mitten in anderen Aufgaben
gestoppt
• Keine Transparenz: Der Anfragende weiß nicht, wo seine Anfrage steht
